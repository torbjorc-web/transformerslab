Exploring Transformers Lab: Carbon Footprint
This project explores text generation with DistilGPT-2 and tracks the carbon footprint of generation runs using CodeCarbon.

Overview
The notebook demonstrates how to:

load the DistilGPT-2 tokenizer and model,
encode prompts into PyTorch tensors,
generate text with greedy search, beam search, n-gram penalties, and sampling,
track emissions with CodeCarbon,
load the generated emissions.csv file with Pandas,
visualize emissions over time.
DistilGPT-2 is a smaller, faster version of GPT-2, and CodeCarbon is a tool for measuring the CO2 impact of machine learning workloads [web:39][web:1].

CodeCarbon’s documentation shows that it can be used with transformer workloads and produces an emissions.csv file with fields like timestamp, duration, emissions, and energy use [web:1].

Project Goals
Generate text with DistilGPT-2.
Compare decoder strategies such as greedy search, beam search, and sampling.
Measure and inspect emissions from model inference.
Plot emissions against runtime duration.
Main Files
notebook.ipynb — the main Codecademy notebook.
emissions.csv — generated by CodeCarbon after running the tracked function.
Suggested Workflow
Install dependencies.
Load the tokenizer and model.
Complete the tokenization and generation tasks.
Add the @track_emissions decorator to the generation function.
Run the function to create emissions.csv.
Read the CSV with Pandas and create the scatter plot.
Example Setup
from transformers import GPT2LMHeadModel, GPT2Tokenizer, set_seed
from codecarbon import track_emissions
import pandas as pd
import matplotlib.pyplot as plt

tokenizer = GPT2Tokenizer.from_pretrained('distilgpt2')

Emissions Tracking
A typical CodeCarbon workflow wraps the generation function with @track_emissions, runs the function once, and then reads the resulting emissions.csv file [web:30][web:43].

Output Visualization
The final plot uses duration on the x-axis and emissions on the y-axis to show how emissions change as runtime increases.
